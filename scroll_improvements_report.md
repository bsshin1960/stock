## 2. 주요 개선 내용 상세

### ① 설정 화면 라벨 텍스트 변경 및 대칭화
* **문제점**: 최초 라디오 버튼 옵션 명칭인 `"스크롤바 적용"`과 `"스크롤바 미적용 & 마우스 휠로 스크롤"`이 대칭성이 부족하고 길어 복잡해 보였습니다.
* **해결책**: 최종적으로 `"스크롤바 적용"`과 **`"휠스크롤 적용"`**으로 변경하여 대칭성을 살리고 직관성을 대폭 향상했습니다.
* **관련 코드**: [src/app.py](./src/app.py#L1563)

### ② 대시보드 휠 스크롤 시 개별 박스 1줄 단위 커스텀 스크롤 고정
* **문제점**: 휠스크롤 적용 모드일 때 개별 정보 박스들(TOP 10, KODEX 200)이 네이티브 스크롤(휠 1칸당 2~3줄씩 이동)로 작동하여 세밀한 컨트롤이 어려웠습니다.
* **해결책**: 모드와 관계없이 항상 각 박스가 휠 1칸당 **정확히 1줄(17px, 21px)**씩 이동하는 인덱스 기반의 정밀 스크롤 레이아웃을 사용하도록 통합 제어했습니다.

### ③ 화면 들썩임(Flicker) 및 스크롤 원위치 점프 현상 완벽 조치
* **문제점**: 
  1. 마우스가 박스에 들어가고 나갈 때 상단 메뉴 드롭다운을 닫는 hack 코드로 인해 전체 페이지가 약 30px가량 상하로 흔들렸습니다.
  2. 스크롤 잠금을 위해 Flet의 부모 Column의 `scroll` 속성을 동적으로 `AUTO`와 `None`으로 계속 토글하는 과정에서 스크롤 뷰가 해제/재구축되면서 스크롤 위치가 무조건 페이지 최상단(0)으로 튕기는 치명적인 버그가 있었습니다.
* **해결책**:
  1. 메뉴가 열려있을 때만 메뉴 드롭다운 닫기 트리거를 작동시키도록 수정하여 무분별한 갱신을 차단했습니다.
  2. 부모 스크롤 위젯의 파괴/재생성을 완전히 피하기 위해, 스크롤 레이아웃을 100% 동일하게 유지한 상태에서 휠 이벤트 콜백 내에 상태값(`is_box_hovered`)에 따라 스크롤 처리를 무시(Lock)하는 **Pure Python Scroll Lock** 아키텍처를 도입하여 들썩임과 튕김 현상을 0%로 완벽 차단했습니다.

### ④ 수평 스크롤바 스타일 개선 (투명도 및 위치 조정)
* **문제점**: AI 의견 카드 아래의 수평 스크롤바가 눈에 너무 띄고, 트랙 좌우 여백으로 인해 스크롤바가 강제로 중앙 정렬되어 스크롤 위치를 오인하기 쉬웠습니다.
* **해결책**: 
  1. 스크롤바 트랙 여백을 초기화(`main_axis_margin=0.0`)하여 스크롤바 핸들이 자연스럽게 가장 왼쪽 끝에서 시작하도록 개선했습니다.
  2. 스크롤바 색상을 클릭 여부와 관계없이 항상 반투명 회색으로 통일하고, 알파(투명도) 값을 50% 추가 증대하여(불투명도 약 12~15% 수준) 배경에 매우 조화롭게 녹아들도록 변경했습니다.
* **관련 코드**: [src/app.py](./src/app.py#L1002-L1015)

### ⑤ 분석 실행 시 전체 스크롤 최상단(0) 자동 초기화
* **문제점**: 분석 완료 후 스크롤을 내려서 결과를 보던 중 새롭게 "분석 실행"을 누르면 여전히 하단에 멈춰있어 상단의 핵심 데이터가 바로 표시되지 않았습니다.
* **해결책**: 분석이 시작되는 시점에 대시보드의 메인 스크롤 좌표 및 모든 개별 박스의 스크롤 인덱스를 `0`으로 일괄 강제 초기화하여 상시 페이지 최상단에서 깔끔하게 시작하도록 개선했습니다.
* **관련 코드**: [src/app.py](./src/app.py#L1815-L1830)

### ⑥ 대시보드 휠 스크롤 속도 대폭 상향 (9배)
* **문제점**: 대시보드 페이지가 길어 휠 한 칸당 `20.0px`씩 스크롤되는 감도는 화면 이동 속도가 너무 답답하게 느껴졌습니다.
* **해결책**: 휠 1칸당 스크롤바 이동 계수를 기존 `20.0px`에서 **`180.0px`**로 총 9배 과감하게 상향 조정하여 매우 빠른 기동성을 확보했습니다.
* **관련 코드**: [src/app.py](./src/app.py#L2083)

### ⑦ 창 확대 시 대시보드 우측 여백 및 일반 리스트 스크롤 차단 문제 수정
* **문제점**:
  1. 전체 창 가로폭이 넓어지면 우측 빈 여백에서는 스크롤 영역(`1297px`) 밖에 마우스가 놓여 휠 감지가 안 되는 구조였습니다.
  2. 대시보드 스크롤을 차단하기 위해 마우스 감지기를 최상위에 전면 배치하면서 내부의 일반 ListView 영역(실시간 뉴스, 소문, 로그, 적중률)의 휠 입력도 함께 막히는 스크롤 불가 현상이 확인되었습니다.
* **해결책**:
  1. 감지용 스택과 컨테이너의 너비를 창의 가로폭(`page_w`)에 유연하게 반응해 가득 차도록 동적으로 반응형 확장했습니다.
  2. 스크롤이 차단되던 4가지 개별 목록 상자들에 대해서도 개별 `GestureDetector`와 python 스크롤 릴레이 함수([handle_news_wheel 등](./src/app.py#L1440))를 설계하여, 마우스 오버 시 대시보드 스크롤은 잠그고 **내부의 목록 컨텐츠만 깔끔하게 개별적으로 상대 스크롤(`scroll_to(delta=...)`)** 되도록 일관성 있게 일체화시켰습니다.
